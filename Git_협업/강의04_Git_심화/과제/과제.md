# 강의04 과제 — 커밋 히스토리 정리

## 과제 목표

`git reset`으로 여러 커밋을 하나로 합치고, `git tag`로 버전을 표시합니다.

## 시나리오

기능 개발 도중 "WIP", "임시 저장" 등의 지저분한 커밋 메시지가 쌓였습니다.  
이를 정리해 깔끔한 커밋 하나로 만들고, 버전 태그를 붙입니다.

## 초기 커밋 히스토리 만들기

```bash
mkdir git-advanced-practice
cd git-advanced-practice
git init

# 파일들 생성 및 지저분한 커밋 쌓기
echo "# 쇼핑몰 프로젝트" > README.md
git add README.md
git commit -m "init: 프로젝트 초기화"

echo "상품 목록 HTML" > products.html
git add products.html
git commit -m "WIP: 상품 목록 작업 중"

echo ".title { color: blue; }" > style.css
git add style.css
git commit -m "임시 저장"

echo "const getProducts = () => [];" > products.js
git add products.js
git commit -m "WIP2: 함수 추가"

echo "const filterProducts = (q) => [];" >> products.js
git add products.js
git commit -m "완성!"
```

현재 히스토리:
```
완성!
WIP2: 함수 추가
임시 저장
WIP: 상품 목록 작업 중
init: 프로젝트 초기화
```

## 과제 요구사항

### 1. 커밋 합치기 (git reset --soft)

가장 최근 4개 커밋(init 제외)을 하나의 의미 있는 커밋으로 합치세요:

```
결과:
feat: 상품 목록 페이지 구현 (HTML/CSS/JS)
init: 프로젝트 초기화
```

힌트:
```bash
git reset --soft HEAD~4   # 4개 커밋을 취소하고 변경사항을 스테이징 유지
git commit -m "feat: 상품 목록 페이지 구현 (HTML/CSS/JS)"
```

### 2. 태그 붙이기

정리된 커밋에 버전 태그를 붙이세요:

```bash
git tag -a v1.0.0 -m "상품 목록 기능 첫 번째 릴리즈"
```

### 3. 확인

```bash
git log --oneline
# abc1234 (HEAD -> main, tag: v1.0.0) feat: 상품 목록 페이지 구현
# def5678 init: 프로젝트 초기화

git show v1.0.0  # 태그 상세 정보
git tag          # 태그 목록
```
