# 강의03 과제 정답 — 오픈소스 기여 흐름 단계별 설명

## 오픈소스 기여 전체 흐름

```
원본 저장소 (upstream)
    ↓ Fork
내 저장소 (origin/fork)
    ↓ clone
로컬 머신
    ↓ 브랜치 생성 + 작업 + 커밋
    ↓ push → origin
    ↓ PR 생성 (origin → upstream)
    ↓ 리뷰 + 승인
    ↓ upstream에 merge
    ↓ 내 fork를 upstream과 동기화
```

---

## 단계 1: 저장소 설정 (로컬 시뮬레이션)

```bash
# 1-1. 원본 저장소 역할 (upstream) 생성
mkdir pr-practice-upstream
cd pr-practice-upstream
git init
echo "# PR 연습 저장소

이 저장소는 Pull Request 연습을 위한 예제입니다." > README.md
git add README.md
git commit -m "init: 저장소 초기화"

# 브랜치 확인
git branch
# * main

cd ..
```

---

## 단계 2: Fork — 내 복사본 만들기

### GitHub에서 Fork할 때:
1. 원본 저장소 페이지에서 `Fork` 버튼 클릭
2. 내 계정 선택
3. 자동으로 `github.com/내아이디/저장소명` 생성

### 로컬 시뮬레이션:

```bash
# upstream 저장소를 복제해 fork 시뮬레이션
git clone pr-practice-upstream pr-practice-fork
cd pr-practice-fork

# origin은 내 fork, upstream은 원본 저장소
# (GitHub에서는 clone 시 origin이 자동으로 내 fork로 설정됨)
git remote add upstream ../pr-practice-upstream

# 리모트 확인
git remote -v
# origin    ../pr-practice-upstream (fetch)  ← clone 경로
# origin    ../pr-practice-upstream (push)
# upstream  ../pr-practice-upstream (fetch)
# upstream  ../pr-practice-upstream (push)
```

## 단계 3: 기능 브랜치에서 작업

```bash
# main 브랜치가 최신인지 확인
git status
# On branch main, nothing to commit

# 기능 브랜치 생성 — 네이밍: feat/기능명, fix/버그명, docs/문서명
git switch -c feature/add-contribution-guide
# Switched to a new branch 'feature/add-contribution-guide'

# 기여 가이드 파일 작성
cat > CONTRIBUTING.md << 'EOF'
# 기여 가이드

이 프로젝트에 기여해 주셔서 감사합니다!

## 기여 방법

### 1. 저장소 Fork
GitHub에서 Fork 버튼을 클릭해 자신의 계정에 복사본을 만드세요.

### 2. 로컬 설정
\`\`\`bash
git clone https://github.com/내아이디/저장소명.git
cd 저장소명
git remote add upstream https://github.com/원본계정/저장소명.git
\`\`\`

### 3. 브랜치 생성
\`\`\`bash
git switch -c feature/내기능이름
\`\`\`

### 4. 커밋 메시지 규칙
- `feat:` 새로운 기능
- `fix:` 버그 수정
- `docs:` 문서 변경
- `refactor:` 코드 개선

### 5. PR 생성
- 제목: `타입: 변경 내용 요약`
- 본문: 변경 이유, 테스트 방법 포함

## 코드 스타일
- 들여쓰기: 2 spaces
- 세미콜론: 필수
- 최대 줄 길이: 100자
EOF

git add CONTRIBUTING.md
git commit -m "docs: 기여 가이드 문서 추가"
# [feature/add-contribution-guide abc1234] docs: 기여 가이드 문서 추가

# 커밋 확인
git log --oneline
# abc1234 (HEAD -> feature/add-contribution-guide) docs: 기여 가이드 문서 추가
# def5678 (origin/main, main) init: 저장소 초기화
```

## 단계 4: Push 및 PR 생성

```bash
# 내 fork(origin)에 브랜치 push
git push origin feature/add-contribution-guide
# Enumerating objects: 4, done.
# To ../pr-practice-fork
#  * [new branch]      feature/add-contribution-guide -> feature/add-contribution-guide

# ── GitHub에서 PR 생성 ──────────────────────────────────────
# push 후 GitHub에 자동으로 "Compare & pull request" 배너가 표시됨
# 또는 저장소 → Pull Requests → New pull request
```

### 작성한 PR 내용 예시

```
제목: docs: 기여 가이드 문서 추가

## Summary
이 프로젝트에 기여하려는 사람들이 참고할 수 있는 CONTRIBUTING.md를 추가했습니다.

## Changes
- CONTRIBUTING.md 신규 추가
- Fork, 브랜치 생성, 커밋 메시지 규칙, PR 생성 방법 안내
- 코드 스타일 가이드 포함

## Type of Change
- [x] 📝 docs: 문서 수정

## Test
- [x] 파일이 정상적으로 렌더링되는지 GitHub 미리보기에서 확인
- [x] 링크 및 코드 블록 정상 표시 확인

## Related Issue
없음 (자발적 기여)
```

## 단계 5: 리뷰 대응

```bash
# 리뷰어가 "커밋 메시지 예시를 추가해달라"고 요청했다고 가정

# 같은 브랜치에서 수정사항 추가
cat >> CONTRIBUTING.md << 'EOF'

## 커밋 메시지 예시
\`\`\`
feat: 사용자 프로필 아바타 업로드 기능 추가
fix: 모바일에서 로그인 버튼 미작동 문제 수정
docs: API 응답 형식 설명 추가
\`\`\`
EOF

git add CONTRIBUTING.md
git commit -m "docs: 커밋 메시지 예시 추가 (리뷰 반영)"

# push — 같은 브랜치에 push하면 PR이 자동으로 업데이트됨
git push origin feature/add-contribution-guide
```

## 단계 6: upstream 동기화

```bash
# upstream(원본)에 새로운 변경사항이 생겼다고 가정
cd ../pr-practice-upstream
echo "
## 라이선스
MIT License" >> README.md
git add README.md
git commit -m "docs: 라이선스 정보 추가"

# 내 fork로 돌아와서 upstream 변경사항 가져오기
cd ../pr-practice-fork
git fetch upstream           # upstream의 변경사항 가져오기 (병합 안 함)
git switch main

# 방법 1: merge로 통합
git merge upstream/main
# 출력: Updating def5678..xyz9999
# Fast-forward
#  README.md | 3 +++

# 방법 2: rebase로 통합 (히스토리 더 깔끔)
# git rebase upstream/main

# 내 fork의 origin에도 업데이트
git push origin main

git log --oneline
# xyz9999 (HEAD -> main, upstream/main, origin/main) docs: 라이선스 정보 추가
# def5678 init: 저장소 초기화
```

## 전체 명령어 요약 (치트시트)

```bash
# 최초 설정
git clone https://github.com/내아이디/저장소.git
cd 저장소
git remote add upstream https://github.com/원본계정/저장소.git

# 기여 시작 시
git fetch upstream
git switch main
git merge upstream/main          # 최신 상태 동기화
git switch -c feature/내기능    # 브랜치 생성

# 작업 후
git add .
git commit -m "feat: 기능 추가"
git push origin feature/내기능  # GitHub에 push → PR 생성

# 리뷰 반영 후 (같은 브랜치에 추가 커밋)
git add .
git commit -m "fix: 리뷰 반영"
git push origin feature/내기능  # PR 자동 업데이트

# PR merge 완료 후 정리
git switch main
git fetch upstream
git merge upstream/main
git branch -d feature/내기능    # 로컬 브랜치 삭제
git push origin --delete feature/내기능  # 원격 브랜치 삭제
```
