# 강의01 과제 — 두 기능 브랜치 동시 개발

## 과제 목표

`feature/header`와 `feature/footer` 두 브랜치를 각각 만들고,  
독립적으로 작업한 뒤 순서대로 `main`에 병합합니다.

## 시나리오

팀에서 두 명이 동시에 작업합니다:
- 개발자 A: `feature/header` — 헤더 네비게이션 구현
- 개발자 B: `feature/footer` — 푸터 정보 구현

혼자 두 역할을 순서대로 진행합니다.

## 단계

### 1단계: 준비

새 디렉토리를 만들고 Git 초기화 후 `index.html` 파일을 만들어 첫 커밋을 합니다.

```html
<!-- index.html 초기 내용 -->
<!DOCTYPE html>
<html>
  <body>
    <main>
      <p>메인 콘텐츠</p>
    </main>
  </body>
</html>
```

### 2단계: feature/header 작업

- `main`에서 `feature/header` 브랜치를 생성합니다.
- `index.html`에 `<header>` 태그와 네비게이션을 추가하는 커밋을 합니다.
- 커밋 메시지: `feat: 헤더 네비게이션 추가`

### 3단계: feature/footer 작업

- `main`으로 돌아와서 `feature/footer` 브랜치를 생성합니다.
- `index.html`에 `<footer>` 태그와 저작권 정보를 추가하는 커밋을 합니다.
- 커밋 메시지: `feat: 푸터 정보 추가`

### 4단계: 순서대로 병합

1. `main`으로 이동
2. `feature/header` 병합
3. `feature/footer` 병합 (충돌이 생길 수 있습니다 — 강의02 내용!)
4. 두 브랜치 모두 삭제
