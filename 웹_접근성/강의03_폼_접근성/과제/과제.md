# 과제03 — 검색 폼 접근성 개선

## 목표

접근성이 부족한 검색 폼을 개선하여 키보드와 스크린리더를 완전히 지원하도록 만듭니다.

---

## 개선 전 코드 (이 코드를 개선하세요)

```html
<!DOCTYPE html>
<html>
<head>
    <style>
        input { border: 1px solid #ccc; padding: 8px; }
        button { background: blue; color: white; padding: 8px 16px; border: none; }
        .error { color: red; }
        input:focus { outline: none; }  /* 포커스 스타일 제거 — 나쁜 예! */
    </style>
</head>
<body>
    <div>
        <input type="text" placeholder="검색어">
        <button onclick="search()">🔍</button>
    </div>
    <div class="error" id="err"></div>
    <ul id="results"></ul>

    <script>
        function search() {
            const q = document.querySelector('input').value;
            if (!q) {
                document.getElementById('err').textContent = '검색어를 입력하세요';
            } else {
                document.getElementById('results').innerHTML =
                    '<li>' + q + '에 대한 결과입니다.</li>';
            }
        }
    </script>
</body>
</html>
```

---

## 개선해야 할 항목 (7가지)

1. **`<html lang="ko">`** 추가
2. **레이블**: `<input>`에 `aria-label` 또는 `<label>` 추가
3. **버튼 텍스트**: 아이콘만 있는 버튼에 `aria-label` 추가
4. **포커스 스타일**: `outline: none` 제거, 명확한 포커스 스타일 추가
5. **에러 메시지**: `role="alert"` + `aria-describedby` 연결
6. **검색 결과**: 결과 업데이트 시 스크린리더 알림 (`aria-live`)
7. **색상 대비**: 버튼, 에러 텍스트, 테두리 색상 대비 4.5:1 이상
