# 과제 03 — 쇼핑몰 타입 설계

## 목표

쇼핑몰 시스템의 핵심 데이터 구조를 TypeScript 인터페이스로 설계하세요.

---

## 설계할 인터페이스

### 1. Product (상품)

```ts
interface Product {
  id: number;
  name: string;
  price: number;
  // TODO: 나머지 필드를 설계하세요
}
```

필수 포함 항목:
- `sku`: 재고 코드 (변경 불가 / readonly)
- `category`: 상품 카테고리
- `stock`: 재고 수량
- `isAvailable`: 판매 여부 (boolean)
- `discountRate?`: 할인율 (선택적, 0~100)
- `imageUrls?`: 이미지 URL 배열 (선택적)

### 2. CartItem (장바구니 항목)

```ts
interface CartItem {
  product: Product;    // Product 인터페이스 활용
  quantity: number;
  // TODO: 나머지 필드를 설계하세요
}
```

필수 포함 항목:
- `selectedOptions?`: 옵션 (색상, 사이즈 등) — 인덱스 시그니처 사용

### 3. Order (주문)

```ts
interface Order {
  id: number;
  // TODO: 나머지 필드를 설계하세요
}
```

필수 포함 항목:
- `items`: CartItem 배열
- `status`: 리터럴 유니온 타입 ('pending' | 'paid' | 'shipping' | 'delivered' | 'cancelled')
- `totalAmount`: 총 금액
- `shippingAddress`: 배송지 (별도 인터페이스로 정의)
- `orderedAt`: 주문 시각 (string, ISO 형식)


---

## 더미 데이터

설계한 인터페이스로 상품 3개, 장바구니 2개 항목, 주문 1개를 만드세요.

---

## 도전 과제

- `ShippingAddress` 인터페이스에 `zipCode` (우편번호) 추가
- `OrderSummary` 타입 — Order에서 id, status, totalAmount만 추출 (`Pick` 사용)
- 할인 금액 계산 함수 추가
