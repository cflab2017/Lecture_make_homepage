# 과제 06 — 타입 안전한 회원가입 폼

## 목표

모든 상태와 이벤트에 TypeScript 타입을 완전히 지정한 회원가입 폼 컴포넌트를 만드세요.

---

## 요구사항

### 폼 필드

| 필드 | 타입 | 유효성 검사 |
|------|------|------------|
| 이름 | text | 2자 이상 |
| 이메일 | email | 올바른 이메일 형식 |
| 비밀번호 | password | 8자 이상 |
| 비밀번호 확인 | password | 비밀번호와 일치 |
| 생년월일 | date | 14세 이상 |
| 성별 | select | 남성/여성/기타/선택 안 함 |
| 이용약관 동의 | checkbox | 필수 동의 |

### 타입 명세

```ts
// 폼 데이터 인터페이스
interface SignupFormData {
  name: string;
  email: string;
  password: string;
  confirmPassword: string;
  birthDate: string;
  gender: 'male' | 'female' | 'other' | '';
  agreeToTerms: boolean;
}

// 유효성 오류 타입 (각 필드별 오류 메시지)
type FormErrors = Partial<Record<keyof SignupFormData, string>>;
```

### 필수 구현 사항

1. 모든 `useState`에 타입 명시
2. 모든 이벤트 핸들러에 올바른 이벤트 타입 사용
   - text/email/password: `React.ChangeEvent<HTMLInputElement>`
   - select: `React.ChangeEvent<HTMLSelectElement>`
   - checkbox: `React.ChangeEvent<HTMLInputElement>`
   - form submit: `React.FormEvent<HTMLFormElement>`
3. 실시간 유효성 검사 (각 필드 blur 시)
4. 제출 시 전체 유효성 검사
5. 성공 시 입력 데이터 콘솔 출력

## 힌트

```ts
// 폼 상태 초기값
const initialFormData: SignupFormData = {
  name: '',
  email: '',
  password: '',
  confirmPassword: '',
  birthDate: '',
  gender: '',
  agreeToTerms: false,
};

// 이메일 유효성 검사
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
emailRegex.test(email); // true / false

// 나이 계산
const age = new Date().getFullYear() - new Date(birthDate).getFullYear();
```
