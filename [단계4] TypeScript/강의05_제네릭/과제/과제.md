# 과제 05 — 제네릭 API 응답 타입

## 목표

`ApiResponse<T>` 제네릭 인터페이스를 만들어, 성공과 실패 응답을 통일된 형식으로 관리하세요.

---

## 요구사항

### ApiResponse<T> 인터페이스

```ts
interface ApiResponse<T> {
  success: boolean;
  data?: T;
  error?: string;
  message?: string;
  timestamp: string;
}
```

### 구현할 헬퍼 함수

```ts
// 성공 응답 생성
function success<T>(data: T, message?: string): ApiResponse<T> { ... }

// 실패 응답 생성
function failure<T = never>(error: string): ApiResponse<T> { ... }
```

### 사용 예시

```ts
// 유저 조회 성공
const userRes = success<User>({ id: 1, name: '홍길동' }, '유저 조회 성공');
// { success: true, data: { id: 1, name: '홍길동' }, message: '유저 조회 성공', timestamp: '...' }

// 유저 목록
const listRes = success<User[]>([...users]);

// 실패 응답
const errRes = failure('유저를 찾을 수 없습니다.');
// { success: false, error: '유저를 찾을 수 없습니다.', timestamp: '...' }
```

---

## 구현할 타입들

```ts
// 페이지네이션 응답
interface PaginatedData<T> {
  items: T[];
  total: number;
  page: number;
  limit: number;
  totalPages: number;
}

// 페이지네이션 API 응답
type PaginatedResponse<T> = ApiResponse<PaginatedData<T>>;
```

---

## 시나리오별 응답 테스트

다음 상황의 응답을 모두 만들어 출력하세요:

1. 유저 1명 조회 성공
2. 유저 목록 (페이지네이션)
3. 상품 목록 조회 성공
4. 존재하지 않는 유저 조회 실패
5. 서버 오류

---

## 도전 과제

- `isSuccess(response: ApiResponse<T>): response is ApiResponse<T> & { data: T }` — 타입 가드 구현
- `unwrap<T>(response: ApiResponse<T>): T` — 성공 시 data 반환, 실패 시 오류 던지기
