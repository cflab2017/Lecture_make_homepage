# 과제04 — 블로그 포스트 API

## 목표

better-sqlite3와 Express를 사용하여 블로그 포스트 CRUD API를 만듭니다.

---

## 요구사항

### 데이터 구조

포스트(post) 테이블에 아래 컬럼을 포함하세요.

| 컬럼 | 타입 | 조건 | 설명 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTOINCREMENT | 고유 번호 |
| `title` | TEXT | NOT NULL | 포스트 제목 |
| `content` | TEXT | NOT NULL | 포스트 내용 |
| `author` | TEXT | NOT NULL | 작성자 |
| `created_at` | TEXT | DEFAULT datetime('now') | 작성일시 |
| `updated_at` | TEXT | DEFAULT datetime('now') | 수정일시 |

### API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/posts` | 전체 포스트 목록 (최신 순) |
| GET | `/posts/:id` | 특정 포스트 조회 |
| POST | `/posts` | 새 포스트 작성 |
| PUT | `/posts/:id` | 포스트 수정 |
| DELETE | `/posts/:id` | 포스트 삭제 |

---

## 구현 조건

1. `better-sqlite3`를 사용할 것
2. 앱 시작 시 `posts` 테이블이 없으면 자동 생성
3. 각 API에서 에러 처리 포함 (없는 ID, 필수 필드 누락)
4. POST `/posts` 요청 시 필수 필드: `title`, `content`, `author`
5. PUT `/posts/:id` 수정 시 `updated_at`을 현재 시각으로 갱신

---

## 테스트 시나리오

```bash
# 1. 포스트 작성
curl -X POST http://localhost:3000/posts \
  -H "Content-Type: application/json" \
  -d '{"title":"첫 번째 글","content":"내용입니다.","author":"김철수"}'

# 2. 전체 목록 조회
curl http://localhost:3000/posts

# 3. 특정 포스트 조회
curl http://localhost:3000/posts/1

# 4. 포스트 수정
curl -X PUT http://localhost:3000/posts/1 \
  -H "Content-Type: application/json" \
  -d '{"title":"수정된 제목"}'

# 5. 포스트 삭제
curl -X DELETE http://localhost:3000/posts/1
```

---

## 추가 도전 과제 (선택)

- `GET /posts?author=김철수` — 작성자로 필터링
- `GET /posts?search=키워드` — 제목에서 키워드 검색
