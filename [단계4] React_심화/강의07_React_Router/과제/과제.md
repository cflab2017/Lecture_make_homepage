# 강의07 과제 — 인증 보호 라우트 (Protected Route)

## 과제 목표

로그인하지 않은 사용자가 보호된 페이지에 접근하면 `/login`으로 리다이렉트하는 **ProtectedRoute 컴포넌트**를 구현하세요.

## 요구사항

### 라우트 구성

```
/           → HomePage (모두 접근 가능)
/login      → LoginPage (로그인 폼)
/dashboard  → DashboardPage (로그인 필요)
/profile    → ProfilePage (로그인 필요)
*           → 404 페이지
```

### ProtectedRoute 컴포넌트

```jsx
// 로그인 상태가 아니면 /login으로 리다이렉트
function ProtectedRoute({ children }) {
  const isLoggedIn = /* 로그인 상태 확인 */;

  if (!isLoggedIn) {
    return <Navigate to="/login" replace />;
  }

  return children;
}
```

### 로그인 상태 관리

실제 API 없이 간단하게 구현하세요:
- `useState`로 로그인 상태(`isLoggedIn`) 관리
- 로그인 폼: 아이디 `admin`, 비밀번호 `1234` 입력 시 로그인 성공
- 로그아웃 버튼 클릭 시 상태를 `false`로 변경

### 동작 시나리오

1. 로그아웃 상태에서 `/dashboard`로 이동 → `/login`으로 자동 리다이렉트
2. 로그인 성공 → `/dashboard`로 이동
3. 헤더의 "로그아웃" 클릭 → 상태 초기화 후 `/`으로 이동
4. 로그인 상태에서 `/login` 접근 → `/dashboard`로 리다이렉트 (역방향 보호)

## 힌트

```jsx
// 로그인 후 원래 가려던 페이지로 돌아가기 (보너스)
// useLocation으로 state에 이전 경로 저장
const location = useLocation();
<Navigate to="/login" state={{ from: location }} replace />

// LoginPage에서 돌아갈 경로 확인
const location = useLocation();
const from = location.state?.from?.pathname || '/dashboard';
navigate(from, { replace: true });
```
