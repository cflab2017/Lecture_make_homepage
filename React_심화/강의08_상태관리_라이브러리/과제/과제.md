# 강의08 과제 — 좋아요 기능 전역 상태

## 과제 목표

**여러 컴포넌트에서 같은 좋아요 수를 공유**하는 전역 상태를 Zustand로 구현하세요.

## 요구사항

### store 구조

```js
// likeStore.js
{
  likes: {
    post1: 0,
    post2: 0,
    post3: 0,
  },
  likedPosts: [],     // 내가 좋아요 누른 게시물 ID 목록

  // 액션
  toggleLike: (postId) => ...,  // 좋아요/취소 토글
  getLikeCount: (postId) => ..., // 특정 게시물의 좋아요 수
  isLiked: (postId) => ...,     // 내가 좋아요 눌렀는지 여부
}
```

### 동작 조건

1. 좋아요 버튼을 누르면 해당 게시물의 좋아요 수가 1 증가한다.
2. 이미 좋아요를 누른 상태에서 다시 누르면 취소 (수 감소).
3. 좋아요 버튼의 색상이 토글 상태에 따라 변한다 (빨간색 ↔ 회색).
4. **같은 게시물의 좋아요 수가 여러 곳에서 동시에 업데이트**된다.

### 컴포넌트 구조

```
App
├── PostFeed (게시물 목록 — LikeButton 포함)
│   ├── PostCard (post1) — LikeButton
│   ├── PostCard (post2) — LikeButton
│   └── PostCard (post3) — LikeButton
└── LikeSummary (각 게시물의 좋아요 수 요약 표시)
```

`LikeSummary`와 `LikeButton`이 같은 좋아요 수를 공유해야 합니다.

## 핵심 포인트

- `likeStore`에서 좋아요 데이터를 관리하면 Context 없이도 모든 컴포넌트가 같은 값을 봅니다.
- 한 곳에서 좋아요를 누르면 `LikeSummary`도 즉시 업데이트됩니다.
