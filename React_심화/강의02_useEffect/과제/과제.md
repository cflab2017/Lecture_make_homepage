# 강의02 과제 — 실시간 검색 (디바운스 없는 버전)

## 과제 목표

`input`에 텍스트를 입력할 때마다 JSONPlaceholder API를 호출해 사용자를 검색하는 컴포넌트를 만드세요.

## 요구사항

### 동작 설명

1. `<input>` 에 검색어를 입력한다.
2. 입력값이 바뀔 때마다 `useEffect`가 실행된다.
3. 빈 문자열이면 API를 호출하지 않고 결과 목록을 비운다.
4. 검색어가 있으면 `https://jsonplaceholder.typicode.com/users` 에서 전체 목록을 가져와 **클라이언트에서 필터링**한다.
5. 로딩 중일 때 "검색 중..." 텍스트를 표시한다.
6. 결과가 없으면 "검색 결과가 없습니다." 를 표시한다.

### API 사용 방법

JSONPlaceholder는 쿼리 파라미터 검색을 완벽하게 지원하지 않습니다.  
전체 목록을 한 번 가져온 뒤 **name 또는 email 필드로 includes 필터링**하세요.

```js
// 전체 사용자 가져오기
const res = await fetch('https://jsonplaceholder.typicode.com/users');
const allUsers = await res.json();

// 클라이언트 필터링
const filtered = allUsers.filter(user =>
  user.name.toLowerCase().includes(검색어.toLowerCase())
);
```

### 필수 state

| state | 타입 | 설명 |
|-------|------|------|
| `query` | `string` | 검색어 |
| `results` | `array` | 필터링된 사용자 목록 |
| `isLoading` | `boolean` | API 호출 중 여부 |
| `error` | `string \| null` | 에러 메시지 |

### useEffect 의존성 배열

`query`가 바뀔 때마다 실행되어야 합니다: `[query]`

## 힌트

- `query`가 빈 문자열(`''`)이면 early return으로 API 호출을 건너뜁니다.
- 검색어가 바뀌면 이전 결과를 지우고 새로 가져옵니다.
- 로딩/에러/결과 세 가지 상태를 조건부 렌더링으로 처리합니다.

## 보너스 (선택)

- 검색어 앞뒤 공백을 `trim()`으로 제거하세요.
- 에러 발생 시 "다시 시도" 버튼을 제공하세요.
