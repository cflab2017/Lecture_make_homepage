# 강의04 과제 — 폼 상태 통합 관리

## 과제 목표

`useReducer`를 사용해 회원가입 폼의 **모든 상태를 하나의 reducer로 통합 관리**하는 컴포넌트를 만드세요.

## 요구사항

### 폼 필드

| 필드 | 타입 | 검증 조건 |
|------|------|----------|
| `name` | text | 2자 이상 |
| `email` | email | '@' 포함 |
| `password` | password | 8자 이상 |
| `confirmPassword` | password | password와 동일 |

### state 구조

```js
const initialState = {
  values: {
    name: '',
    email: '',
    password: '',
    confirmPassword: '',
  },
  errors: {
    name: '',
    email: '',
    password: '',
    confirmPassword: '',
  },
  isSubmitting: false,
  isSubmitted: false,
};
```

### action 목록

| action type | payload | 설명 |
|------------|---------|------|
| `SET_FIELD` | `{ field, value }` | 특정 필드 값 변경 |
| `SET_ERROR` | `{ field, message }` | 특정 필드 에러 메시지 설정 |
| `SUBMIT_START` | 없음 | 제출 시작 (isSubmitting: true) |
| `SUBMIT_SUCCESS` | 없음 | 제출 완료 (isSubmitted: true) |
| `RESET` | 없음 | 전체 초기화 |

### 동작 조건

1. 입력 시 해당 필드의 에러를 즉시 초기화한다.
2. 제출 버튼 클릭 시 모든 필드를 검증한다.
3. 에러가 있으면 제출하지 않고 에러 메시지를 표시한다.
4. 모든 검증 통과 시 `isSubmitting: true`로 설정하고 2초 후 `isSubmitted: true`가 된다.
5. 제출 완료 후 "가입 완료!" 메시지와 "처음으로" 버튼을 보여준다.

## 힌트

```jsx
// SET_FIELD reducer 처리 예시
case 'SET_FIELD':
  return {
    ...state,
    values: {
      ...state.values,
      [action.payload.field]: action.payload.value, // 계산된 프로퍼티 키
    },
    errors: {
      ...state.errors,
      [action.payload.field]: '', // 해당 필드 에러 초기화
    },
  };
```
