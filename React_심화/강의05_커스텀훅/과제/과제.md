# 강의05 과제 — useDebounce 훅 구현

## 과제 목표

검색어 입력 후 **500ms 뒤에** API를 호출하는 `useDebounce` 커스텀 훅을 구현하세요.

## 디바운스(Debounce)란?

연속된 이벤트 중 **마지막 이벤트가 발생한 후** 일정 시간이 지나야 실제 처리를 실행하는 기법입니다.

```
키 입력:    a → ab → abc → abcd
타이머:     |   |    |     |--- 500ms ---> API 호출!
결과:       타이머가 계속 초기화되다가 마지막 입력 후 500ms만에 호출
```

## 요구사항

### useDebounce 훅

```js
// 파일: useDebounce.js
function useDebounce(value, delay = 500) {
  // value가 delay ms 동안 변하지 않으면 debouncedValue를 업데이트
  // ...
  return debouncedValue;
}
```

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| `value` | `any` | 디바운스할 값 (검색어 등) |
| `delay` | `number` | 대기 시간(ms), 기본값 500 |

### 사용 예시

```jsx
function SearchComponent() {
  const [query, setQuery] = useState('');
  const debouncedQuery = useDebounce(query, 500); // 500ms 후에 업데이트

  // debouncedQuery가 바뀔 때만 API 호출 (입력할 때마다 호출 X)
  useEffect(() => {
    if (debouncedQuery) {
      fetchSearchResults(debouncedQuery);
    }
  }, [debouncedQuery]);

  return <input value={query} onChange={e => setQuery(e.target.value)} />;
}
```

### 구현 원리

```
1. value가 바뀌면 setTimeout으로 delay ms 후에 debouncedValue 업데이트
2. value가 delay 이내에 다시 바뀌면 clearTimeout으로 이전 타이머 취소
3. 최종적으로 delay 동안 값이 유지되면 debouncedValue가 업데이트됨
```

## 결과 확인

`useDebounce`를 사용해 검색 컴포넌트를 만들고 다음을 확인하세요:

- 빠르게 타이핑할 때 API가 호출되지 않는다.
- 타이핑을 멈추고 500ms 후에 API가 호출된다.
- 브라우저 Network 탭에서 호출 횟수가 줄어드는 것을 확인한다.
