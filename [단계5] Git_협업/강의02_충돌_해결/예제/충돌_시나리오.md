# 의도적으로 충돌 만들고 해결하는 단계별 실습

## 목표

충돌이 어떻게 생기는지 직접 경험하고, 해결하는 절차를 익힙니다.

---

## 준비

```bash
# 새 실습 디렉토리
mkdir conflict-practice
cd conflict-practice
git init
git config user.name "실습자"
git config user.email "test@example.com"

# 공통 기반 파일 생성
cat > style.css << 'EOF'
/* 공통 스타일 */
body {
  font-family: sans-serif;
  margin: 0;
  padding: 0;
}

button {
  color: black;
  background-color: gray;
  border: none;
  padding: 8px 16px;
}

header {
  height: 60px;
  background-color: white;
}
EOF

git add style.css
git commit -m "init: 기본 스타일 설정"
```

---

## 단계 1: 두 브랜치 생성 및 같은 파일 수정

```bash
# ── 브랜치 A: feature/theme-blue ─────────────────────────
git switch -c feature/theme-blue

# style.css에서 button 색상을 파란색으로 변경
cat > style.css << 'EOF'
/* 공통 스타일 */
body {
  font-family: sans-serif;
  margin: 0;
  padding: 0;
}

button {
  color: white;
  background-color: #3b82f6;  /* 파란색으로 변경 (theme-blue) */
  border: none;
  padding: 8px 16px;
  border-radius: 6px;         /* 새로 추가 */
}

header {
  height: 60px;
  background-color: white;
}
EOF

git add style.css
git commit -m "style: 버튼을 파란색 테마로 변경"

# ── main으로 돌아와서 브랜치 B 생성 ──────────────────────
git switch main

# style.css에서 button 색상을 빨간색으로 변경 (다른 팀원 작업 시뮬레이션)
cat > style.css << 'EOF'
/* 공통 스타일 */
body {
  font-family: sans-serif;
  margin: 0;
  padding: 0;
}

button {
  color: white;
  background-color: #ef4444;  /* 빨간색으로 변경 (main) */
  border: none;
  padding: 8px 16px;
}

header {
  height: 60px;
  background-color: #1e293b;  /* 헤더도 변경 */
}
EOF

git add style.css
git commit -m "style: 버튼 빨간색, 헤더 다크 테마로 변경"

# 현재 분기 구조 확인
git log --oneline --graph --all
# * xxxxxxx (HEAD -> main) style: 버튼 빨간색, 헤더 다크 테마로 변경
# | * yyyyyyy (feature/theme-blue) style: 버튼을 파란색 테마로 변경
# |/
# * zzzzzzz init: 기본 스타일 설정
```

---

## 단계 2: 의도적으로 충돌 발생시키기

```bash
# main에서 feature/theme-blue 병합 시도
git merge feature/theme-blue

# ── 충돌 발생! ────────────────────────────────────────────
# 출력:
# Auto-merging style.css
# CONFLICT (content): Merge conflict in style.css
# Automatic merge failed; fix conflicts and then commit the result.

# 충돌 파일 확인
git status
# On branch main
# You have unmerged paths.
#   (fix conflicts and run "git commit")
#   (use "git merge --abort" to abort the merge)
#
# Unmerged paths:
#   (use "git add <file>..." to mark resolution)
#         both modified:   style.css    ← 충돌 파일
```

---

## 단계 3: 충돌 마커 확인

```bash
# 충돌 파일 내용 확인
cat style.css
```

충돌이 발생한 `style.css`는 다음과 같이 보입니다:

```css
/* 공통 스타일 */
body {
  font-family: sans-serif;
  margin: 0;
  padding: 0;
}

button {
  color: white;
<<<<<<< HEAD
  background-color: #ef4444;  /* 빨간색으로 변경 (main) */
  border: none;
  padding: 8px 16px;
}

header {
  height: 60px;
  background-color: #1e293b;  /* 헤더도 변경 */
=======
  background-color: #3b82f6;  /* 파란색으로 변경 (theme-blue) */
  border: none;
  padding: 8px 16px;
  border-radius: 6px;         /* 새로 추가 */
}

header {
  height: 60px;
  background-color: white;
>>>>>>> feature/theme-blue
}
```

### 마커 해석

| 마커 | 의미 |
|------|------|
| `<<<<<<< HEAD` | 현재 브랜치(main)의 내용 시작 |
| `=======` | 두 내용을 구분하는 선 |
| `>>>>>>> feature/theme-blue` | 병합하려는 브랜치의 내용 끝 |

---

## 단계 4: 충돌 해결

팀 논의 결과: "파란색 버튼 + 다크 헤더"로 결정했다고 가정합니다.

```bash
# style.css를 원하는 내용으로 수정 (마커를 모두 제거해야 함!)
cat > style.css << 'EOF'
/* 공통 스타일 */
body {
  font-family: sans-serif;
  margin: 0;
  padding: 0;
}

button {
  color: white;
  background-color: #3b82f6;  /* 팀 결정: 파란색 */
  border: none;
  padding: 8px 16px;
  border-radius: 6px;         /* feature/theme-blue에서 추가된 내용 유지 */
}

header {
  height: 60px;
  background-color: #1e293b;  /* main에서 변경된 다크 헤더 유지 */
}
EOF

# ⚠️ 주의: <<<<<<< / ======= / >>>>>>> 마커가 하나도 없어야 함!
# 마커가 남아있으면 CSS가 깨짐
```

---

## 단계 5: 해결 완료 및 병합 커밋

```bash
# 충돌 해결된 파일을 스테이징
git add style.css

# 스테이징 상태 확인
git status
# On branch main
# All conflicts fixed but you are still merging.
#   (use "git commit" to conclude merge)
#
# Changes to be committed:
#         modified:   style.css

# 병합 커밋 생성
git commit -m "merge: feature/theme-blue 병합 (파란 버튼 + 다크 헤더 채택)"
# [main abc1234] merge: feature/theme-blue 병합

# 최종 히스토리 확인
git log --oneline --graph --all
# *   abc1234 (HEAD -> main) merge: feature/theme-blue 병합
# |\
# | * yyyyyyy (feature/theme-blue) style: 버튼을 파란색 테마로 변경
# * | xxxxxxx style: 버튼 빨간색, 헤더 다크 테마로 변경
# |/
# * zzzzzzz init: 기본 스타일 설정

# 병합된 브랜치 삭제
git branch -d feature/theme-blue
```

---

## 보너스: 충돌 없이 자동 병합되는 경우

```bash
# 다른 파일을 수정하거나 같은 파일의 다른 부분을 수정하면
# Git이 자동으로 병합해줌 → 충돌 없음!

# 예: main에서 body를 수정, feature에서 header만 수정
# → 자동 병합 가능
```

## 체크리스트

- [ ] 의도적으로 두 브랜치에서 같은 파일의 같은 부분을 수정했다.
- [ ] 충돌 마커(`<<<<<<<`, `=======`, `>>>>>>>`)를 모두 찾았다.
- [ ] 마커를 제거하고 원하는 내용으로 수동 편집했다.
- [ ] `git add` 후 `git commit`으로 병합을 완료했다.
- [ ] `git log --oneline --graph`로 병합 커밋을 확인했다.
