# 강의02 과제 정답 — 충돌 해결 과정

## 전체 명령어 및 설명

### 1단계: 초기 설정

```bash
mkdir conflict-assignment
cd conflict-assignment
git init
git config user.name "홍길동"
git config user.email "hong@example.com"

# 공통 기반 파일 생성
cat > config.js << 'EOF'
// config.js
const config = {
  appName: '팀 프로젝트',
  version: '1.0.0',
  theme: 'default',
  language: 'ko',
  debugMode: false,
};

export default config;
EOF

git add config.js
git commit -m "init: 기본 설정 파일 추가"
# [main (root-commit) aaa1111] init: 기본 설정 파일 추가
```

### 2단계: feature/settings-update 브랜치 (팀원 A)

```bash
# main에서 브랜치 생성
git switch -c feature/settings-update
# Switched to a new branch 'feature/settings-update'

# config.js 수정 — theme을 dark로, apiUrl 추가
cat > config.js << 'EOF'
// config.js
const config = {
  appName: '팀 프로젝트',
  version: '1.0.0',
  theme: 'dark',                                  // 변경: default → dark
  language: 'ko',
  debugMode: false,
  apiUrl: 'https://api.example.com',              // 추가
};

export default config;
EOF

git add config.js
git commit -m "feat: 다크 테마 설정 및 API URL 추가"
# [feature/settings-update bbb2222] feat: 다크 테마 설정 및 API URL 추가
```

### 3단계: main으로 돌아와서 팀원 B 작업

```bash
# main으로 이동 (config.js가 원래 상태로 돌아옴)
git switch main
# Switched to branch 'main'

# config.js의 현재 내용 확인 — 원래 버전이어야 함
cat config.js
# theme: 'default' (feature/settings-update의 변경사항 없음)

# main에서 직접 수정 (팀원 B 작업 시뮬레이션)
cat > config.js << 'EOF'
// config.js
const config = {
  appName: '팀 프로젝트',
  version: '1.0.0',
  theme: 'light',      // 변경: default → light (팀원 A와 같은 줄 수정!)
  language: 'en',      // 변경: ko → en
  debugMode: false,
};

export default config;
EOF

git add config.js
git commit -m "feat: 라이트 테마 및 영어 설정"
# [main ccc3333] feat: 라이트 테마 및 영어 설정

# 분기 구조 확인
git log --oneline --graph --all
# * ccc3333 (HEAD -> main) feat: 라이트 테마 및 영어 설정
# | * bbb2222 (feature/settings-update) feat: 다크 테마 설정 및 API URL 추가
# |/
# * aaa1111 init: 기본 설정 파일 추가
```

### 4단계: 충돌 발생시키기

```bash
# main에서 feature/settings-update 병합 시도
git merge feature/settings-update

# ── 충돌 발생! ────────────────────────────────────────────
# 출력:
# Auto-merging config.js
# CONFLICT (content): Merge conflict in config.js
# Automatic merge failed; fix conflicts and then commit the result.

# 충돌 상태 확인
git status
# both modified:   config.js
```

### 5단계: 충돌 내용 분석

```bash
cat config.js
```

충돌 마커가 포함된 파일 내용:

```js
// config.js
const config = {
  appName: '팀 프로젝트',
  version: '1.0.0',
<<<<<<< HEAD
  theme: 'light',      // 변경: default → light (팀원 B)
  language: 'en',      // 변경: ko → en (팀원 B)
  debugMode: false,
};
=======
  theme: 'dark',                                  // 변경: default → dark (팀원 A)
  language: 'ko',
  debugMode: false,
  apiUrl: 'https://api.example.com',              // 추가 (팀원 A)
};
>>>>>>> feature/settings-update

export default config;
```

**마커 해석:**
- `<<<<<<< HEAD` ~ `=======`: main(팀원 B)의 변경사항 — `light` 테마, `en` 언어
- `=======` ~ `>>>>>>> feature/settings-update`: 팀원 A의 변경사항 — `dark` 테마, `apiUrl` 추가
- `theme` 줄에서 양쪽이 다르게 수정했으므로 충돌 발생
- `language`는 한쪽만 수정했지만 같은 블록에 포함되어 충돌 영역에 포함됨

### 6단계: 충돌 해결

팀 합의: `theme: 'system'`, `language: 'en'`, `apiUrl` 모두 유지

```bash
# config.js를 원하는 내용으로 직접 작성 (마커 완전 제거)
cat > config.js << 'EOF'
// config.js
const config = {
  appName: '팀 프로젝트',
  version: '1.0.0',
  theme: 'system',                                // 팀 합의: system (OS 테마)
  language: 'en',                                 // 팀원 B 변경 유지
  debugMode: false,
  apiUrl: 'https://api.example.com',              // 팀원 A 추가 유지
};

export default config;
EOF

# ⚠️ 마커(<<<<<<< / ======= / >>>>>>>) 가 없는지 반드시 확인
grep -c "<<<<<<" config.js
# 출력: 0  (마커 없음 — OK)
```

### 7단계: 충돌 해결 완료

```bash
# 해결된 파일을 스테이징
git add config.js

# 상태 확인
git status
# All conflicts fixed but you are still merging.
# Changes to be committed: modified: config.js

# 병합 커밋 생성
git commit -m "merge: settings-update 병합 (theme=system, en 언어, apiUrl 유지)"
# [main ddd4444] merge: settings-update 병합

# 최종 히스토리
git log --oneline --graph
# *   ddd4444 (HEAD -> main) merge: settings-update 병합
# |\
# | * bbb2222 feat: 다크 테마 설정 및 API URL 추가
# * | ccc3333 feat: 라이트 테마 및 영어 설정
# |/
# * aaa1111 init: 기본 설정 파일 추가

# 최종 파일 확인
cat config.js
# theme: 'system', language: 'en', apiUrl 모두 있어야 함

# 브랜치 삭제
git branch -d feature/settings-update
```

---

## 충돌 해결 핵심 절차 요약

```
1. 충돌 발생 (git merge)
      ↓
2. git status로 충돌 파일 확인
      ↓
3. 파일 열어서 마커 찾기
      ↓
4. 마커 제거 + 원하는 내용으로 편집
      ↓
5. git add <파일>
      ↓
6. git commit
      ↓
7. 완료!
```

## 중간에 포기하고 싶을 때

```bash
# 병합 전 상태로 완전히 돌아가기
git merge --abort

# merge --abort 이후: 마치 merge를 시도하지 않은 상태가 됨
```
