# 강의03 과제 — 오픈소스 기여 흐름 시뮬레이션

## 과제 목표

오픈소스 프로젝트에 기여하는 **fork → branch → commit → PR** 흐름을 직접 경험합니다.

## 시나리오

GitHub에 있는 팀 저장소(`upstream`)에 기여하는 과정을 시뮬레이션합니다.

## 단계

### 1단계: 저장소 준비 (강사가 원본 저장소 역할)

GitHub에 빈 저장소를 하나 만드세요 (이것이 `upstream`):
- 저장소 이름: `pr-practice`
- README.md 초기화 체크

```bash
# 또는 로컬에서 origin 역할을 할 저장소를 시뮬레이션
mkdir upstream-repo && cd upstream-repo
git init
echo "# PR 연습 저장소" > README.md
git add . && git commit -m "init: 저장소 초기화"
```

### 2단계: Fork (자신의 복사본 만들기)

GitHub에서 Fork 버튼 클릭 → 자신의 계정에 복사본 생성

```bash
# 로컬 시뮬레이션
cd ..
git clone upstream-repo my-fork
cd my-fork
git remote add upstream ../upstream-repo
git remote -v
# origin   ../my-fork (fetch)
# upstream ../upstream-repo (fetch)
```

### 3단계: 기능 브랜치에서 작업

```bash
git switch -c feature/add-contribution-guide

# 새 파일 추가
echo "# 기여 가이드" > CONTRIBUTING.md
git add CONTRIBUTING.md
git commit -m "docs: 기여 가이드 문서 추가"
```

### 4단계: Push 및 PR 생성

```bash
git push origin feature/add-contribution-guide
```

GitHub에서 PR 생성 시 다음 내용을 포함하세요:

| 항목 | 내용 |
|------|------|
| 제목 | `docs: 기여 가이드 문서 추가` |
| Summary | 기여 방법을 안내하는 CONTRIBUTING.md 추가 |
| Type | docs |
| Test | - [ ] 파일 존재 여부 확인 |

### 5단계: upstream 업데이트 후 동기화

```bash
# upstream에 새 커밋이 생겼다고 가정
cd ../upstream-repo
echo "## 코드 스타일" >> README.md
git add . && git commit -m "docs: 코드 스타일 섹션 추가"

# fork에 upstream 변경사항 동기화
cd ../my-fork
git fetch upstream
git switch main
git merge upstream/main   # 또는 git rebase upstream/main
```
