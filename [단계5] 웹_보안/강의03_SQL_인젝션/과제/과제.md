# 과제03 — 검색 기능 SQL 인젝션 방어

## 목표

취약한 검색 쿼리를 Prepared Statement로 안전하게 변환합니다.

---

## 취약한 코드 (이 코드를 수정하세요)

```js
// vulnerable-search.js
const express  = require('express');
const Database = require('better-sqlite3');

const app = express();
const db  = new Database(':memory:');

db.exec(`
    CREATE TABLE products (id INTEGER PRIMARY KEY, name TEXT, price INTEGER, category TEXT);
    INSERT INTO products VALUES (1, '노트북', 1500000, '전자제품');
    INSERT INTO products VALUES (2, '마우스',   30000, '전자제품');
    INSERT INTO products VALUES (3, '클린코드', 32000, '도서');
`);

// ⚠ 취약점: 쿼리 문자열에 사용자 입력을 직접 삽입
app.get('/search', (req, res) => {
    const keyword = req.query.q || '';
    // SQL 인젝션 취약점!
    const results = db.prepare(
        `SELECT * FROM products WHERE name LIKE '%${keyword}%'`
    ).all();
    res.json(results);
});

app.listen(3000);
```

---

## 테스트할 공격 페이로드 (방어 후 정상 결과만 반환해야 함)

```
GET /search?q=노트북              → 정상 검색
GET /search?q=' OR '1'='1        → 모든 상품 반환되면 안 됨
GET /search?q='; DROP TABLE --   → 에러 없이 정상 처리
GET /search?q=%' UNION SELECT... → 다른 테이블 정보 반환 안 됨
```

---

## 구현 조건

1. Prepared Statement + `?` 플레이스홀더 사용
2. 입력값 길이 제한 (최대 100자)
3. 에러 시 서버 에러 정보를 클라이언트에 노출하지 않음
4. 서버 로그에는 에러 기록
