# 과제04 — 환경변수로 API 키 분리

## 목표

하드코딩된 API 키와 설정값을 `.env` 파일로 안전하게 분리합니다.

---

## 취약한 코드 (이 코드를 수정하세요)

```js
// app.js — 시크릿이 하드코딩된 위험한 코드
const express  = require('express');
const mongoose = require('mongoose');

const app = express();

// ⚠ 시크릿이 코드에 직접 노출됨
const DB_PASSWORD = 'mySecretPassword123';
const MONGODB_URI = `mongodb+srv://admin:${DB_PASSWORD}@cluster0.abc.mongodb.net/mydb`;
const JWT_SECRET  = 'jwt-secret-key-do-not-share';
const API_KEY     = 'sk-abcdefghijk12345678';
const PORT        = 3000;

mongoose.connect(MONGODB_URI);

app.get('/data', async (req, res) => {
    // API_KEY를 헤더로 외부 서비스 호출 (하드코딩)
    const response = await fetch('https://api.example.com/data', {
        headers: { 'Authorization': `Bearer ${API_KEY}` }
    });
    res.json(await response.json());
});

app.listen(PORT);
```

---

## 구현 조건

1. `.env` 파일에 모든 시크릿 이동
2. `.env.example` 파일 생성 (키 이름만, 실제 값 없이)
3. `.gitignore`에 `.env` 추가
4. `dotenv` 패키지로 환경변수 로드
5. 필수 환경변수 누락 시 서버 시작 거부 (검증 함수)

---

- `과제정답/.env.example` — 환경변수 템플릿
