# 과제 — 카드 컴포넌트 만들기

## 목표

박스 모델(padding, border, margin)을 정확하게 이해하고,  
실무에서 자주 사용하는 **프로필 카드 컴포넌트**를 만듭니다.

---

## 요구사항

### 카드 구조 (HTML)

```html
<div class="card">
  <div class="card-image">
    <!-- 이미지 또는 배경색으로 대체 -->
  </div>
  <div class="card-body">
    <h2 class="card-name">홍길동</h2>
    <p class="card-role">프론트엔드 개발자</p>
    <p class="card-desc">안녕하세요! CSS를 공부하고 있습니다.</p>
    <div class="card-tags">
      <span class="tag">HTML</span>
      <span class="tag">CSS</span>
      <span class="tag">JavaScript</span>
    </div>
  </div>
</div>
```

### CSS 조건

| 속성 | 요구사항 |
|------|----------|
| `box-sizing` | `border-box` 전체 적용 |
| `width` | 카드 너비 300px (또는 max-width 활용) |
| `padding` | card-body에 24px 이상 |
| `border` | 카드 전체에 1px 테두리 또는 box-shadow |
| `border-radius` | 카드 모서리 둥글게 (12px 이상) |
| `margin` | 카드를 화면 중앙에 배치 (`margin: auto`) |
| `card-image` | 높이 180px, 배경색으로 이미지 역할 대체 |

### 추가 도전 (선택사항)

- `.tag`를 `display: inline-block`으로 설정하고 padding/border-radius 적용
- 카드에 `:hover` 효과 추가 (그림자가 진해지는 효과)
- 3개의 카드를 나란히 배치해 보세요 (이번 강의 범위를 약간 넘어가지만 도전!)

---

## 체크리스트

- [ ] `* { box-sizing: border-box; }` 를 적용했다
- [ ] 카드가 화면 중앙에 배치된다
- [ ] `padding`과 `margin`이 구분되어 사용되었다
- [ ] `border-radius`로 모서리가 둥글다
- [ ] `.tag`가 나란히 배치된다
- [ ] 인라인 CSS를 사용하지 않았다

---

## 힌트

```css
/* 카드 이미지 영역: 배경색으로 이미지 대체 */
.card-image {
  height: 180px;
  background-color: #3498db; /* 원하는 색상 */
  border-radius: 12px 12px 0 0; /* 위 모서리만 둥글게 */
}

/* 태그: inline-block으로 나란히 배치 */
.tag {
  display: inline-block;
  padding: 4px 10px;
  border-radius: 20px;
  background-color: #eee;
  font-size: 0.8rem;
}
```
