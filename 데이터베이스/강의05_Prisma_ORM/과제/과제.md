# 과제05 — 댓글 기능 추가

## 목표

강의05 예제의 User-Post 구조에 **댓글(Comment)** 기능을 추가합니다.  
Post-Comment 1:N 관계를 스키마에 정의하고, 댓글 CRUD API를 구현합니다.

---

## 요구사항

### 1. 스키마 수정 (`schema.prisma`)

기존 `User`, `Post` 모델에 `Comment` 모델을 추가하세요.

**Comment 모델에 필요한 필드:**

| 필드 | 타입 | 조건 | 설명 |
|------|------|------|------|
| `id` | Int | PK, autoincrement | 고유 번호 |
| `content` | String | NOT NULL | 댓글 내용 |
| `post` | Post | @relation | Post와 N:1 관계 |
| `postId` | Int | | 외래키 |
| `author` | User | @relation | User와 N:1 관계 |
| `authorId` | Int | | 외래키 |
| `createdAt` | DateTime | @default(now()) | 작성일시 |

**관계 추가:**
- `Post` 모델에 `comments Comment[]` 추가
- `User` 모델에 `comments Comment[]` 추가

### 2. 마이그레이션 실행

```bash
npx prisma migrate dev --name add-comment
```

### 3. 댓글 API 구현 (`routes.js`)

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/posts/:postId/comments` | 특정 포스트의 댓글 목록 |
| POST | `/posts/:postId/comments` | 댓글 작성 |
| PUT | `/comments/:id` | 댓글 수정 |
| DELETE | `/comments/:id` | 댓글 삭제 |

---

## 구현 조건

1. 댓글 목록은 **최신 순**으로 정렬
2. 댓글 조회 시 **작성자 이름** 포함
3. 댓글 수정/삭제 시 **존재 여부** 확인 후 처리
4. POST `/posts/:postId/comments` 필수 필드: `content`, `authorId`
