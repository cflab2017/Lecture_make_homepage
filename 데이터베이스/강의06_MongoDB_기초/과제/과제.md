# 과제06 — 태그 기반 메모 앱

## 목표

강의06 예제의 Memo 모델에 **태그(tags) 배열**을 추가하고,  
태그로 메모를 검색하는 기능을 구현합니다.

---

## 요구사항

### 1. 모델 수정 (`models/Memo.js`)

기존 Memo 스키마에 `tags` 필드를 추가하세요.

```js
// 추가할 필드
tags: {
    type:    [String],   // 문자열 배열
    default: [],         // 기본값: 빈 배열
}
```

### 2. API 수정 (`routes/memo.js`)

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/memos` | 전체 메모 목록 |
| GET | `/memos?tag=태그명` | 특정 태그의 메모 조회 |
| GET | `/memos/tags` | 사용 중인 전체 태그 목록 |
| GET | `/memos/:id` | 단일 메모 조회 |
| POST | `/memos` | 메모 생성 (tags 배열 포함 가능) |
| PUT | `/memos/:id` | 메모 수정 (tags 수정 포함) |
| DELETE | `/memos/:id` | 메모 삭제 |

---

## 구현 조건

### 태그 검색

```
GET /memos?tag=공부
```
- `tags` 배열에 해당 태그가 포함된 메모만 반환

### 태그 목록 조회

```
GET /memos/tags
```
- 모든 메모에서 사용된 태그를 **중복 없이** 반환
- MongoDB의 `distinct()` 메서드 활용

```js
const tags = await Memo.distinct('tags');
// ['개발', '공부', '일상', '여행'] 형태로 반환
```

## 예시 요청/응답

```bash
# 메모 생성 (태그 포함)
POST /memos
{
    "title": "MongoDB 공부",
    "content": "find, findById, create...",
    "tags": ["공부", "개발", "MongoDB"]
}

# 태그로 검색
GET /memos?tag=개발
# → tags 배열에 '개발'이 포함된 메모만 반환

# 전체 태그 목록
GET /memos/tags
# → ["MongoDB", "개발", "공부", "일상"]
```
