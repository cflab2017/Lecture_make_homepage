# 과제01 — 쇼핑몰 DB 설계

## 목표

간단한 쇼핑몰 데이터베이스 구조를 설계합니다.  
실제 코드 작성 전에 **어떤 테이블이 필요한지, 어떤 관계가 있는지** 먼저 생각하는 연습입니다.

---

## 요구사항

다음 기능을 가진 쇼핑몰의 데이터베이스를 설계하세요.

- 회원 가입 및 로그인
- 상품 목록 조회
- 상품 주문 (한 주문에 여러 상품 가능)
- 주문 내역 조회

---

## 과제 내용

### 1. 필요한 테이블 목록

아래 빈칸을 채우세요.

```
필요한 테이블:
1. __________ 테이블 (회원 정보)
2. __________ 테이블 (상품 정보)
3. __________ 테이블 (주문 정보)
4. __________ 테이블 (주문-상품 연결, 한 주문에 여러 상품)
```

### 2. 각 테이블의 컬럼 설계

각 테이블에 필요한 컬럼을 텍스트로 표현하세요.

예시 형식:
```
테이블명 (컬럼1: 타입, 컬럼2: 타입, ...)
```

```
users      (?, ?, ?, ?)
products   (?, ?, ?, ?)
orders     (?, ?, ?)
order_items(?, ?, ?, ?)
```

### 3. 텍스트 ERD 작성

테이블 간의 관계를 텍스트로 표현하세요.

```
users ─── orders        (관계: 한 회원은 여러 주문 가능)
orders ─── order_items  (관계: ?)
products ─── order_items (관계: ?)
```

### 4. 기본키와 외래키 표시

각 테이블에서 PRIMARY KEY와 FOREIGN KEY를 표시하세요.

---

## 제출 형식

`과제정답/schema.sql` 파일에 실제 CREATE TABLE 문으로 작성하세요.

---

## 힌트

- 주문 하나에 상품 여러 개 → **중간 테이블(order_items)** 필요
- 회원과 주문의 연결 → `user_id` 외래키
- 가격은 주문 시점 가격을 저장해야 함 (나중에 상품 가격이 바뀔 수 있음)
