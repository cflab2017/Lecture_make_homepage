# Lecture_make_homepage

HTML/CSS/Git을 처음 배우는 완전 초보자를 위한 **홈페이지 만들기 특강 시리즈**입니다.
한 강의당 1~2시간 분량이며, 각 강의를 따라하면 진짜 작동하는 내 홈페이지가 만들어집니다.

---

## 강의 목록

### [강의 1. GitHub + Vercel로 5분 만에 내 홈페이지 만들기](./강의1)

> **결과물:** `https://내이름.vercel.app` 형태의 진짜 작동하는 내 홈페이지

서버, FTP, 도메인 같은 복잡한 절차 없이 **GitHub에 코드를 올리기만 하면 Vercel이 자동으로 배포**해주는 현대적인 방식을 배웁니다.

- GitHub, Vercel 계정 만들기
- 첫 HTML 파일 작성하기
- GitHub 저장소에 업로드 → Vercel 자동 배포
- 수정 → 커밋 → 자동 재배포 흐름 체험

📁 [강의 자료](./강의1/README.md) · [예제 코드](./강의1/src)

### [강의 2. CSS로 내 홈페이지 예쁘게 꾸미기](./강의2)

> **선수 조건:** 강의1 완료 (배포된 `index.html`이 있는 상태)

강의1에서 만든 페이지를 **색상, 폰트, 레이아웃**으로 꾸며봅니다. CSS의 핵심 문법과 Flexbox, 반응형까지 입문자가 꼭 알아야 할 내용을 다룹니다.

- CSS 기본 문법 (선택자, 속성, 값)
- 색상·폰트·여백·테두리
- Flexbox로 레이아웃 잡기
- 미디어 쿼리로 반응형 구현

📁 [강의 자료](./강의2/README.md) · [예제 코드](./강의2/src)

### [강의 3. Git 명령어로 진짜 개발자처럼 일하기](./강의3)

> **선수 조건:** 강의1·2 완료 (GitHub 웹에서 클릭으로 업로드해본 경험)

지금까지는 GitHub 웹사이트에서 클릭으로 파일을 올렸지만, 이번엔 **터미널에서 직접 Git 명령어로** 코드를 관리합니다. 진짜 개발자가 매일 쓰는 워크플로입니다.

- Git 설치, 사용자 정보, PAT 인증 설정
- `clone`으로 저장소 내려받기
- 핵심 4명령: `status` → `add` → `commit` → `push`
- `log`, `diff`로 변경 이력 추적
- 충돌, 실수 되돌리기 등 자주 만나는 문제 해결

📁 [강의 자료](./강의3/README.md) · [치트시트 & 예제](./강의3/src)

### [강의 4. JavaScript로 내 홈페이지에 동작 넣기](./강의4)

> **선수 조건:** 강의1~3 완료

지금까지의 페이지는 "사진"이었다면, 이번엔 **"영상"으로 만듭니다.** 버튼을 누르면 반응하고, 입력에 따라 실시간으로 바뀌는 살아있는 홈페이지를 직접 만들어봅니다.

- JavaScript의 정체와 HTML/CSS와의 차이
- 변수, 데이터 타입, 연산 기초
- DOM 조작 — 요소 찾기 / 변경하기
- 이벤트(`click`, `input`)로 사용자 동작에 반응
- 조건문·반복문·함수
- 미니 프로젝트: **다크모드 토글, 클릭 카운터, 실시간 인사**

📁 [강의 자료](./강의4/README.md) · [예제 코드](./강의4/src)

### [강의 5. React + Next.js로 진짜 웹앱 만들기](./강의5)

> **선수 조건:** 강의1~4 완료

지금까지 만든 정적 홈페이지를 넘어, **현업에서 실제로 쓰이는 React + Next.js**로 진짜 웹앱을 만들어 Vercel에 배포합니다. 5강 시리즈의 마지막이자 클라이맥스.

- React/Next.js의 정체와 컴포넌트 기반 사고
- Node.js 환경 구축, `create-next-app`으로 프로젝트 생성
- 컴포넌트와 props로 부품 조립
- `useState`로 상태 관리
- 폴더 기반 라우팅 (`/about` 페이지 추가)
- Next.js 앱을 Vercel에 배포

📁 [강의 자료](./강의5/README.md) · [예제 코드](./강의5/src)
