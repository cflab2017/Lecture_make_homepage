# 과제04 — 포트폴리오 접근성 전체 점검

## 목표

이미지, 폼, 색상 대비, 키보드 탐색 항목을 체계적으로 점검하는 방법을 익힙니다.

---

## 점검 항목 (4개 영역)

### 영역 1. 이미지 점검

- [ ] 모든 `<img>`에 `alt` 속성 있는지
- [ ] 장식용 이미지는 `alt=""` 처리했는지
- [ ] 정보 전달 이미지의 alt 텍스트가 충분히 구체적인지
- [ ] 링크/버튼 이미지의 alt가 목적지/동작을 설명하는지

### 영역 2. 폼 점검

- [ ] 모든 입력 필드에 `<label>` 또는 `aria-label` 있는지
- [ ] 에러 메시지에 `role="alert"` 있는지
- [ ] 에러 메시지가 입력 필드와 `aria-describedby`로 연결됐는지
- [ ] 필수 필드에 `aria-required="true"` 있는지

### 영역 3. 색상 대비 점검

- [ ] 본문 텍스트 대비율 4.5:1 이상
- [ ] 큰 텍스트(18pt+) 대비율 3:1 이상
- [ ] 버튼, 입력창 테두리 대비율 3:1 이상
- [ ] 색상만으로 정보를 전달하는 곳은 없는지

### 영역 4. 키보드 탐색 점검

- [ ] Tab만으로 모든 인터랙티브 요소 접근 가능한지
- [ ] 포커스 표시(outline)가 명확한지 (outline: none 없는지)
- [ ] 모달/드롭다운이 Escape로 닫히는지
- [ ] Skip Navigation 링크가 있는지 (긴 내비게이션이 있는 경우)

---

## 점검 방법

각 항목에 대해 아래 도구를 사용하세요.

| 영역 | 도구 |
|------|------|
| 이미지 | Lighthouse, 코드 직접 확인 |
| 폼 | axe DevTools, 코드 직접 확인 |
| 색상 대비 | WebAIM Contrast Checker, Chrome DevTools |
| 키보드 | Tab 키 실제 테스트, NVDA/VoiceOver |

---

## 제출 형식

`과제정답/checklist.md` 파일에 아래 형식으로 작성하세요.

```markdown
# 접근성 점검 체크리스트

## 이미지 점검
- [x] 항목: 해결 방법/코드
- [ ] 항목: 미개선 이유

## 폼 점검
...
```

각 항목마다 **해결 코드** 또는 **발견된 문제와 수정 방법**을 작성하세요.
